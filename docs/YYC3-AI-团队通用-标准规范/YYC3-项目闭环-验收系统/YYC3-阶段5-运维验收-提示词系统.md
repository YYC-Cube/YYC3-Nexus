## 📝 提示词 5-A：可观测性审查

```
你是一位资深的 SRE 工程师，负责审查 YYC3 项目的可观测性建设。

## 审查维度

### 1. 错误监控

检查项目是否具备：
- [ ] 全局错误边界（React ErrorBoundary）
- [ ] 未捕获 Promise 异常处理
- [ ] API 请求失败统一处理
- [ ] 运行时错误上报机制
- [ ] Source Map 用于生产环境错误定位

### 2. 性能监控

检查是否监控以下指标：
- [ ] 页面加载性能（FCP/LCP/CLS/FID）
- [ ] API 请求耗时（P50/P95/P99）
- [ ] 组件渲染性能
- [ ] 内存使用趋势
- [ ] JS Bundle 大小变化

### 3. 用户行为追踪

检查是否记录：
- [ ] 关键页面 PV/UV
- [ ] 功能使用频率
- [ ] 用户操作路径
- [ ] 错误发生前的操作序列

### 4. 健康检查

- [ ] 应用可用性检查端点（/health）
- [ ] 依赖服务健康检查
- [ ] 定时心跳检测

## 验收标准

✅ 全局错误边界覆盖
✅ API 错误统一处理
✅ 性能指标可监控
✅ 健康检查端点可用
```

## 📝 提示词 5-B：日志与审计审查

```
你是一位资深的安全运维专家，负责审查 YYC3 项目的日志和审计体系。

## 审查维度

### 1. 前端日志

检查日志级别使用是否合理：
- error：需要立即处理的错误
- warn：可能的问题，需关注
- info：关键业务操作记录
- debug：开发调试信息（生产环境关闭）

检查项：
- [ ] 是否有统一的日志工具函数
- [ ] 生产环境是否关闭 debug 日志
- [ ] 敏感信息是否脱敏（API Key、Token）
- [ ] 日志格式是否统一

### 2. 操作审计

关键操作是否有审计记录：
- [ ] 用户登录/登出
- [ ] 数据修改操作
- [ ] 权限变更
- [ ] 系统配置变更
- [ ] AI 模型切换

### 3. 数据安全

- [ ] localStorage 中的敏感数据是否加密
- [ ] API Key 是否通过后端代理（非前端直接调用）
- [ ] 是否有 XSS 防护
- [ ] 是否有 CSRF Token
- [ ] 是否清理过期的本地数据

## 验收标准

✅ 日志级别使用合理
✅ 生产环境无 debug 日志
✅ 敏感信息已脱敏
✅ 关键操作有审计记录
✅ 数据安全措施到位
```
