## 📖 使用指南

### 快速开始

1. **确定当前阶段**：根据项目进度，确认应执行的验收阶段
2. **打开对应文档**：打开 `YYC3-阶段{N}-{名称}-提示词系统.md`
3. **执行提示词**：按文档顺序逐条执行提示词
4. **记录结果**：每条提示词执行后记录验收结果
5. **生成报告**：使用文档中的报告模板生成验收报告
6. **通过判定**：检查验收清单是否全部通过
7. **进入下阶段**：通过后进入下一阶段

### 基础命令速查

```bash
# 代码质量
pnpm typecheck          # TypeScript 类型检查
pnpm lint               # Biome 代码规范检查
pnpm lint:fix           # Biome 自动修复

# 测试
pnpm test               # 运行所有单元测试
pnpm test:coverage      # 运行测试并生成覆盖率报告
pnpm test:watch         # 监听模式运行测试
pnpm test:e2e           # 运行 E2E 测试

# 构建
pnpm build              # 生产构建
pnpm preview            # 预览构建产物

# 依赖
pnpm install            # 安装依赖
pnpm audit              # 安全审计
```

## 📝 总纲提示词：全量闭环验收执行

````
你是一位资深的项目管理专家和全栈工程师，负责对 YYC3 项目执行全量闭环验收。

## 项目上下文

- 项目：YYC³ AI 营销智能中枢 v1.0.2
- 技术栈：React 18 + TypeScript 5 + Vite 6 + Tailwind CSS 4 + Radix UI + Zustand
- 测试：Vitest 3 + Playwright
- 代码质量：Biome 2

## 执行计划

按以下顺序逐阶段执行验收：

### 第一步：环境准备
```bash
pnpm install
pnpm typecheck
pnpm lint
````

### 第二步：需求验证（阶段0）

1. 遍历 src/app/components/ 梳理功能清单
2. 对比需求文档标记实现状态
3. 生成需求追溯矩阵

### 第三步：架构验证（阶段1）

1. 检查目录结构合规性
2. 审查 package.json 依赖
3. 检查状态管理架构
4. 验证构建配置

### 第四步：编码验证（阶段2）

1. pnpm typecheck 检查类型安全
2. pnpm lint 检查代码规范
3. 审查组件实现质量
4. 检查无障碍合规

### 第五步：测试验证（阶段3）

1. pnpm test 运行所有测试
2. pnpm test:coverage 生成覆盖率
3. 审查测试质量
4. 检查 E2E 覆盖

### 第六步：部署验证（阶段4）

1. pnpm build 构建验证
2. 产物分析
3. 性能基线测试
4. 安全检查

### 第七步：运维验证（阶段5）

1. 检查错误监控
2. 审查日志体系
3. 验证文档完整性
4. 生成闭环总结报告

## 输出要求

为每个阶段生成验收报告，最终生成项目闭环总结报告。
所有报告使用统一的模板格式（见各阶段文档）。
